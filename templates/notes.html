How the Web Works

The web is a set of computers that communicate with each other. To visit page, a computer sends an HTTP Request to a server. The server finds the appropriate HTML document and sends it back to the computer where a browser interprets the page and displays it on the screen. This video explains it.

HTML

HTML stands for Hypertext Markup Language. HTML documents form the majority of the content on the web. HTML documents contain text content which describes "what you see" and markup which describes "how it looks". Overview

Tags and Elements

Variables

Python Lists Lists are the most versatile of Python's compound data types. A list contains items separated by commas and enclosed within square brackets ([]). To some extent, lists are similar to arrays in C. One difference between them is that all the items belonging to a list can be of different data type. The values stored in a list can be accessed using the slice operator ([ ] and [:]) with indexes starting at 0 in the beginning of the list and working their way to end -1. The plus (+) sign is the list concatenation operator, and the asterisk (*) is the repetition operator.

Python Tuples are another sequence data type that are similar to the list. A tuple consists of a number of values separated by commas. Unlike lists, however, tuples are enclosed within parentheses.

The main differences between lists and tuples are: Lists are enclosed in brackets ( [ ] ) and their elements and size can be changed, while tuples are enclosed in parentheses ( ( ) ) and cannot be updated. Tuples can be thought of as read-only lists.

Templates, Strings, Lists, and Dictionaries

Strings are immutable, while lists and dictionaries are mutable. GET and POST parameters are included in the URL. GET parameters are used to fetch documents, while POST params change data on the server. GET parameters are affected by the URL length, while POST params have no max length. GET params may be cached, while POST params almost never are. GETs should never change the server, while POSTS should. Tip: don't use GET requests to alter any data no server, as plugins can hit get links, unintentionally altering data and causing UI/UX problems.

HTML Templating to avoid repetition

HTML templating engines like Jinja help avoid repetition in multiple ways, including allowing page headers and footers to be defined once and used across an entire site. This site usese Jinja in this way even though at the time of this writing, it's only one page. If I add an 'about' page in the future, it will be as easy as extending the base html with { % include "about.html" % } instead of the current { % include "notes.html" % }.


Jinja also allows code to be written insde html files, which came in handy when building the commens section of this site. Rather than building the html for the individual comments using string substitution or string construction, I was able to keep the html within my comments html file, with the addtion of a jinja for loop: 

  { % for post in posts % }
      <blockquote class=ind_responses>
        { % autoescape true % }
            { { post.content } } 
        { % endautoescape % }
      </blockquote>
  { % endfor % }

Input Validation: Security and UX

User input that is not validated can lead to two problems: an unhappy user, and an insecure website. Users who make a mistake entering their email address should be told that they have made a mistake, otherwise there may be no way of contacting them. Similarly, in the comments section of this site, empty comment are no allowed. Users who mistakenly hit 'submit' before typing anything or who intentionally leave a blank comment are instead redirected to the input text area, whose outline changed to red. An error message accompanies appears below the box to prompt the user to enter something before hitting 'submit'.


Input validation is also important for security. A malicious user on an unsecure website can implement a cross-site scripting (XSS) attack, stealing private information like email addresses, passwords, and credit card numbers stored on the site's servers. By escaping user input, attempts at injecting code are instead rendered harmlessly as text.
